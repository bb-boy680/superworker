/skill-creator
我想要优化 wiki 的 skill、让他可以创建一个 Agent Teams 来协同编写文档
原因：由于 wiki 是在一个单一的 Agent 上执行的、不能开启多个 Agent                    
来协同工作、这就会导致一个上下文窗口会溢出、导致文档质量下降
我的诉求：    
- 我需要多个 Agent 来协同工作、不同的 Agent 来执行不同的流程
- 主 Agent、用来管理文档输出进度、如果文档缺失、需要指派 Agent 来生成文档
- 代码检索、模块拆解、我需要这个 Agent 可以检索我的工程代码、并拆分出模块的文档、在 progress.json 记录一个任务    
例如：检索到一个模块需要生成文档、在 progress.json 文件 pending_modules  中插入这个文件路径
我需要模块的拆分、还有功能直接对应上代码
- wiki 生成的 Agent、用来专门处理 wiki的文档生成、他是一个可以能生成企业级别的文档、能把功能对应到代码上、同时还有流程图、时序图来帮助理解
注意：文档要写的足够清晰、就和一个专业级别的文章一样
- 文档质量检测 Agent、用来检测 wiki 生成的 Agent 的结果进行考核、检测质量是否符合企业级、专业级的文章、如果文章质量不行
需要打回给 wiki 生成 Agent来补充内容                                                                                                 

注意：
- 这是一个通用性的 Agent、你不需要查看当前目录的代码、也不要根据当前工程定制化规则 
- 还有文档生成时、不要同时派遣多个 wiki 文档生成 Agent、一个个的来
例如：我要生成第一个文档、派遣一个文档生成 Agent、生成完毕 派遣 文档质量检测Agent、当这个文档通过质量检测时
关闭这个文档的生成 Agent 和质量检测 Agent、然后开始下一个文档的生成


## Step2. wiki 文档生成

如何生成:
- 我需要你深度阅读这个文档的依赖文件、并根据依赖文件生成一个符合企业级、专业级别的 wiki 文档
- Wiki 文档需要有 mermaid 图、marmaid 时序图、和依赖文件
- 结构层级清晰，支持快速检索与二次编辑
每一个层级都添加一个依赖文件说明、方便文件快速检索

例如：
## 依赖文件
| 文件   | 说明                          |
| ---- | --------------------------- |
| `path` | 关于这个文件的详细说明 |

- 需要有完善的上下文和代码示例

相关文档是表格的形式展示

# 相关文档
| 文档   | 说明                          |
| ---- | --------------------------- |
| `path` | 关于这个文档的详细说明 |

注意:
- 全程使用**标准Markdown语法**，Mermaid代码块可直接复制渲染
- 语言专业正式，无口语化表达，符合企业技术文档规范
- 严格依据你提供的依赖文件生成，不编造无依据内容
- 图表逻辑与文档文字完全对应，依赖关系无错误

## Step3. 文档审核

审核标准：
- 有架构设计
- 有 mermaid 图、时序图
- 有层级结构
- 层级结构有依赖文件
- 依赖关系无错误、不编造无依据内容
