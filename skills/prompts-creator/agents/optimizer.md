# 提示词优化 Agent

## 优化流程

```
诊断问题 → 应用原则 → 精简内容 → 测试验证
```

---

## 一、诊断阶段

**读取参考资料**：[anti-patterns.md](references/anti-patterns.md) - 识别反面教材

| 检查项 | 操作 |
|--------|------|
| 识别重复内容 | 同一配置出现几次？ |
| 统计强调词数量 | "必须""严禁"出现几次？ |
| 检查负面指令占比 | "禁止"占比多少？ |
| 检查结构是否清晰 | 能否画成流程图？ |

**诊断输出**：列出发现的问题清单

---

## 二、优化阶段

**读取参考资料**：[quick-reference.md](references/quick-reference.md) - 问题速查解决方案

| 检查项 | 操作 |
|--------|------|
| 核心规则放入第一层 | description 是否有核心规则？ |
| 完整配置只在开头定义 | 定义一次还是多次？ |
| 关键规则在每个使用点提醒 | 有渐进提示吗？ |
| 正面指令替代负面指令 | "禁止X"改成"使用Y"？ |
| 解释"为什么" | 有原因解释吗？ |
| 结构替代强调 | 有流程图/列表吗？ |

**优化输出**：逐项修复问题

---

## 三、验证阶段

| 检查项 | 操作 |
|--------|------|
| 多轮对话测试 | 5+回合后规则仍有效？ |
| 边界情况测试 | 模型能正确判断？ |
| 行数对比 | 优化后是否精简？ |

**验证输出**：确认优化效果

---

## 四、优化检查清单

**诊断后自检**：
- [ ] 是否识别出所有重复内容？
- [ ] 是否统计了强调词数量？
- [ ] 是否检查了负面指令占比？

**优化后自检**：
- [ ] description 是否有核心规则？
- [ ] 完整配置是否只定义一次？
- [ ] 关键位置是否有渐进提醒？
- [ ] 是否用正面指令替代了负面指令？
- [ ] 是否解释了"为什么"？

**验证后自检**：
- [ ] 多轮对话测试是否通过？
- [ ] 优化后是否精简？
