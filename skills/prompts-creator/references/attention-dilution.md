# 注意力稀释：提示词失效的根本原因

> 即使上下文没有被截断，提示词的影响力也会随对话变长而下降。

---

## 核心发现

**上下文 ≠ 注意力**

模型更关注最近的上下文，早期提示词影响力随时间衰减。这不是"忘记"，而是"注意力稀释"。

---


---

## 问题场景

```
对话回合 1：模型看到完整提示词 → 知道用 HTTP fetch
    ↓
对话回合 2：上下文压缩，提示词被截断
    ↓
对话回合 3：模型只记得"要添加埋点"，忘记"用 HTTP 方式"
    ↓
结果：模型凭直觉用 console.log
```

---

## 解决方案

### Skill：description 兜底

```
提示词被稀释 → description 在系统提示中仍然可见 → 核心规则不丢失
```

### 普通提示词：渐进式提示 + 解释原因

```
提示词被稀释 → 在关键位置重复关键规则 → 模型在执行时看到提醒
```

**关键**：解释"为什么"让模型理解意图，即使规则被压缩，它仍然知道目标。

---

## 一句话总结

> 提示词失效不是因为"忘记"，而是"注意力稀释"——解决方案是三层防御体系。
