# 反面教材：常见的错误提示词模式

> 识别这些模式，避免重蹈覆辙。

---

## 一、"打补丁"恶性循环

```
模型忘记规则 → 加一条"必须" → 还是忘记 → 加更多强调词
→ 对话变长后仍忘记 → 每个位置重复规则 → 提示词臃肿冗长
→ 模型更难理解重点 → 回到起点
```

**根本原因**：用"堵"的方式解决问题，而不是找到根本原因。

---

## 二、典型反面教材

### ❌ 过度强调

```markdown
## ⚠️ 核心规则：每次回复后必须调用 AskUserQuestion

**这是本 skill 最重要的规则，必须严格遵守！**

**❌ 禁止行为**：
- ❌ 回复后不调用 AskUserQuestion
- ❌ 使用不同的选项配置
- ❌ 对话变长后"忘记"调用
```

**问题**：过度强调（⚠️、必须、严禁）、负面指令过多、没有解释原因。

### ✅ 正确写法

```markdown
每次回复末尾，用以下配置询问用户下一步：

AskUserQuestion({...})
```

## 三、错误模式速查表

| 错误模式 | 症状 | 正确做法 |
|----------|------|----------|
| 过度强调 | 大量"必须""严禁""⚠️" | 删除强调词，用清晰指令 |
| 负面清单 | 大篇幅写"禁止做什么" | 改为正面指令 |
| 重复冗余 | 同一规则出现 5 次 | 定义 1 次 + 渐进提醒 |
| 抽象概念 | "严禁跳步""步骤锁" | 用流程图表达顺序 |
| 没有原因 | 只说"必须"不说"为什么" | 解释原因 |
| 分散定义 | 规则散落各处 | 集中在开头 + 关键位置提醒 |
