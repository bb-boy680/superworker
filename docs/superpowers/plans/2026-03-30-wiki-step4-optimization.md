# 步骤 4 Wiki 文档生成优化计划

## Context

**问题**：当前步骤 4 是逐个串行生成文档，效率低。且文档类型单一，没有区分目录主文件、固定文件、普通模块文件。

**用户需求**：
1. 批量并行召唤 SubAgent（最多 5 个）
2. 区分文档类型：目录主文件、固定文件、普通模块文件
3. 高质量文档要求：流程图、时序图、架构图、示例代码
4. 客户端/服务端文档分开编写
5. 数据库设计文档（如涉及）

**为什么**：
- 并行生成提升效率，60 个文档串行需要很久
- 不同文档类型需要不同的生成策略
- 高质量文档帮助开发者快速理解系统，也帮助 AI 定位 BUG

## 文档类型分类

### 类型 1：固定文件（4 个）

| 文件 | 内容 | 特殊要求 |
|------|------|---------|
| `快速开始.md` | 项目入门指南 | 环境准备、安装、启动步骤 |
| `架构设计.md` | 系统架构概述 | 全局架构图、技术选型 |
| `开发手册.md` | 开发规范和指南 | 代码规范、提交规范、测试规范 |
| `文档地图.md` | 所有文档索引 | 动态生成，列出所有模块链接 |

### 类型 2：目录主文件

**识别规则**：分支节点（有 children）

**内容结构**：
```markdown
# {模块名称}

## 概述
{模块简介}

## 子模块索引
| 文档 | 说明 |
|------|------|
| [子模块A](./子模块A.md) | ... |
| [子模块B](./子模块B.md) | ... |

## 快速开始
{引导读者从哪个子模块开始}

## 相关文档
- [返回上级](../父模块.md)
- [文档地图](../../文档地图.md)
```

### 类型 3：普通模块文件

**识别规则**：叶子节点（无 children，有 leaf_module_ids）

**内容结构**：
```markdown
# {模块名称}

## 概述
{功能描述、设计目标}

## 架构设计
{架构图 Mermaid}

## 核心流程
{时序图/流程图 Mermaid}

## 实现细节
{核心类/函数说明}

## API 参考
{接口定义}

## 使用示例
{可运行代码}

## 依赖文件
| 文件 | 说明 |
|------|------|

## 相关文档
{父/子/兄弟节点链接}
```

## 批量并行生成策略

### 并发控制

```
每批次最多 5 个 SubAgent 并行
批次策略：按文档类型分组

批次 1: 固定文件（4 个）→ 可并行
批次 2: 目录主文件（按层级从上到下）→ 同层级可并行
批次 3: 普通模块文件 → 最多 5 个并行
```

### 批次调度算法

```javascript
// 伪代码
const nodes = loadNodes(); // 从 progress.json

// 分组
const fixedDocs = ['快速开始', '架构设计', '开发手册', '文档地图'];
const branchDocs = nodes.filter(n => !n.is_leaf); // 目录主文件
const leafDocs = nodes.filter(n => n.is_leaf);    // 普通模块

// 批次执行
await runBatch(fixedDocs, 4);        // 固定文件一批
await runBatch(branchDocs, 5);       // 目录主文件（同层级并行）
await runBatches(leafDocs, 5);       // 普通模块分批执行
```

## 文档生成质量标准

### 必须包含

| 内容 | 类型 | 说明 |
|------|------|------|
| 概述 | 文字 | 150-300 字 |
| 架构图 | Mermaid | graph TD/TB，展示模块关系 |
| 流程图/时序图 | Mermaid | sequenceDiagram/flowchart |
| 核心代码示例 | 代码块 | 可运行、带注释 |
| 依赖文件表格 | 表格 | 列出所有相关源文件 |
| 相关文档链接 | 链接 | 父/子/兄弟节点 |

### 字数要求

| 文档类型 | 最少字数 |
|---------|---------|
| 固定文件 | 1000+ |
| 目录主文件 | 500+ |
| 普通模块文件 | 1500+ |

### 图表要求

| 文档类型 | 图表数量 |
|---------|---------|
| 固定文件 | 1+ |
| 目录主文件 | 0-1（可选） |
| 普通模块文件 | 2+ |

## 客户端/服务端文档分离

### 判断规则

从 `config.yaml` 读取 `frontend` 和 `backend` 匹配规则：

```yaml
frontend:
  - "src/pages/**"
  - "src/components/**"
  - "*.tsx"
  - "*.vue"

backend:
  - "src/controllers/**"
  - "src/services/**"
  - "*.controller.ts"
  - "*.service.ts"
```

### 文档差异

| 方面 | 客户端文档 | 服务端文档 |
|------|-----------|-----------|
| 读者 | 前端开发者 | 后端开发者 |
| 重点 | UI 组件、交互流程、状态管理 | API 接口、数据库、业务逻辑 |
| 示例代码 | React/Vue 组件 | API 调用、SQL 查询 |
| 图表 | 组件树、状态流转 | 数据流、API 时序 |

### 数据库设计文档

**触发条件**：模块包含数据库相关文件（`models/`、`entities/`、`*.sql`）

**内容**：
- 表结构说明
- 字段说明
- 索引设计
- ER 图（Mermaid erDiagram）

## 执行流程（优化后）

```
步骤 4.0: 初始化进度
    ↓
步骤 4.1: 生成固定文件（4 个并行）
    ↓
步骤 4.2: 获取目录主文件批次
    ↓ 有节点
步骤 4.3: 批量生成目录主文件（最多 5 个并行）
    ↓ 回到 4.2
    ↓ 无节点
步骤 4.4: 获取普通模块批次
    ↓ 有节点
步骤 4.5: 批量生成普通模块文档（最多 5 个并行）
    ↓ 回到 4.4
    ↓ 无节点
步骤 4.6: 验证所有文档
    ↓
完成
```

## wiki-generator Agent Prompt 模板

### 目录主文件 Prompt

```
生成目录索引文档。

节点信息：
- name: {name}
- doc_path: {doc_path}
- children: {children_list}

任务：
1. 分析子模块功能
2. 生成子模块索引表格
3. 写入 {doc_path}

内容结构：
# {name}

## 概述
{模块简介，50-100 字}

## 子模块索引
| 文档 | 说明 | 适用场景 |
|------|------|---------|
| [子模块A](./子模块A.md) | ... | ... |

## 快速开始
- 新手建议从 [xxx](./xxx.md) 开始
- ...

## 相关文档
- [返回上级](../xxx.md)
- [文档地图](../../文档地图.md)
```

### 普通模块文件 Prompt

```
生成专业级 Wiki 文档。

节点信息：
- name: {name}
- doc_path: {doc_path}
- paths: {source_files}
- description: {description}
- module_type: frontend | backend | shared

任务：
1. 读取 paths 中所有源文件
2. 分析代码结构，提取核心信息
3. 生成高质量文档：
   - 架构图（Mermaid graph）
   - 流程图/时序图（Mermaid sequence/flowchart）
   - 核心代码示例（可运行、带注释）
   - API 参考
   - 依赖文件表格

质量要求：
- 字数 ≥ 1500
- 图表 ≥ 2 个
- 基于代码不编造
- 示例代码可运行

特殊处理：
- 如果 module_type = backend 且涉及数据库：添加数据库设计章节
- 如果 module_type = frontend：重点介绍组件和交互

完成后写入 {doc_path} 并报告结果。
```

## 验证方法

1. **执行步骤 4**：`/wiki` 触发完整流程
2. **检查并行效率**：观察是否有多个 Agent 同时运行
3. **验证文档质量**：
   - 字数是否达标
   - 图表是否正确
   - 代码示例是否可运行
4. **检查特殊文档**：
   - 固定文件是否存在
   - 目录主文件是否有子模块索引
